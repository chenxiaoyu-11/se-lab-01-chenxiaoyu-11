# 软件工程课程学习计划

## 一、课程目标
通过本课程的学习，我将系统掌握软件工程的核心流程与方法，能够独立完成从需求分析、系统设计、编码实现到测试维护的完整小型软件项目开发，同时熟练使用 Git、IDE、UML 等开发工具，培养规范的工程化开发思维与团队协作能力。

## 二、学习内容与阶段安排
1.  **基础工具阶段（第1-2周）**：完成 Git、GitHub、IDE 及 UML 工具的安装与配置，掌握版本控制、环境搭建与基础建模能力，完成本次实验任务。
2.  **需求与设计阶段（第3-6周）**：学习需求分析方法与 UML 建模技术，掌握用例图、类图、时序图等设计工具，理解面向对象设计原则，完成小型系统的需求与设计文档。
3.  **编码实现阶段（第7-10周）**：结合设计文档，使用 Java/Python 进行模块化编码，遵循编码规范与最佳实践，完成系统功能实现。
4.  **测试与部署阶段（第11-14周）**：学习单元测试、集成测试方法，掌握软件部署与持续集成的基本流程，提升软件质量保障能力。
5.  **项目总结阶段（第15-16周）**：完成课程大作业，整合所学知识进行项目复盘与文档撰写，形成完整的软件工程实践成果。

## 三、学习方法
1.  **实践驱动**：每周完成实验任务，及时将代码与文档提交至 GitHub 仓库，养成版本控制习惯。
2.  **理论结合**：阅读教材与经典软件工程书籍，将理论知识与实际项目结合，加深理解。
3.  **协作交流**：积极参与课堂讨论与小组项目，提升沟通表达与团队协作能力。

## 四、预期成果
1.  完成所有实验任务，GitHub 仓库内容完整、提交规范。
2.  掌握 UML 建模与软件设计能力，能独立完成小型系统的设计与实现。
3.  具备规范的编码与工程化思维，为后续专业学习与工作打下坚实基础。
